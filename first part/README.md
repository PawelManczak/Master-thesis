# Master's Thesis - Part 1: Preliminary Experiments & RDF Processing

This directory contains the preliminary or alternative branch of the master's thesis research, primarily focusing on extracting and analyzing temporal association rules (using the ARMADA algorithm) from RDF data structures.

## Purpose

The main goal of this module is to parse semantic data formats (RDF), clean them, extract temporal intervals, and apply the ARMADA algorithm to find temporal relationships and behavioral/structural patterns.

## Directory Structure

*   **`config.py`**: General configuration parameters for paths and algorithm thresholds.
*   **`run_armada.py`**: The main execution script to run temporal rule mining on processed interval data.
*   **`data/`**: Data storage.
    *   `raw/`: Initial RDF files (e.g., `GraphNeuralNetwork.rdf`) and datasets.
    *   `temp/`: Temporary parsing files and segmented RDF versions.
    *   `processed/`: Cleaned RDFs, extracted CSV data, and temporal intervals ready for rule mining.
*   **`src/`**: Source code containing the analytical pipeline.
    *   `analysis/`: Implementations of `armada_temporal.py` and `rdf_analyzer.py`.
    *   `utils/`: Utility scripts for data manipulation (`csv_extractor.py`, `rdf_cleaner.py`, `temporal_interval_merger.py`, etc.).
    *   `examples/`: Sample scripts demonstrating the entire processing pipeline (`complete_pipeline.py`).
*   **`output/`**: Output generated by the mining algorithms.
    *   `reports/`: Discovered rules and patterns (e.g., `armada_rules_all_intervals...csv`, `armada_patterns_...csv`).
    *   `plots/`: Visualizations of the extracted patterns.

## Usage

To start the temporal rule mining process on the extracted intervals:
```bash
python run_armada.py
```
